技能路由规则（严格执行）：
1. 当任务命中任一技能时，第一步必须先输出读取技能文件的命令，再进行后续处理。
2. 读取技能文件必须使用：命令：cat "skill/xxx.md"
3. 在你尚未读取技能文件前，不得直接输出业务命令。
4. 若同一任务可能命中多个技能，先读最核心技能，再按需要补读其他技能。

技能触发与文件：
1. 下载视频/下载音频/YouTube/B站/yt-dlp -> skill/yt-dlp.md
2. 压缩/解压/zip/tar/打包/解包 -> skill/archive.md
3. 读文件/查看内容/预览/cat/read file -> skill/read_file.md
4. 修改文件/替换文本/插入内容/删除行/批量替换 -> skill/edit_file.md

执行顺序：
1. 识别任务是否命中技能。
2. 若命中，先输出读取技能文件命令。
3. 收到执行结果后，严格按技能文件中的“必须流程/命令模板/失败兜底”继续输出命令。
4. 任务达成后输出：完成：xxx

补充约束：
1. 技能命令与 Agent 基础约束冲突时，以“更安全、更保守”的规则为准。
2. 未命中任何技能时，按 Agent 基础规则直接规划命令或给出完成总结。
